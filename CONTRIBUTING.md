# Contributing

This project is maintained as a learning resource. Contributions, suggestions, and improvements are welcome.

## How to Contribute

1. Fork this repository on GitHub.
2. Create a new branch: `git checkout -b feature/your-feature-name`
3. Make your changes.
4. Commit with a clear message: `git commit -m "Add: description of change"`
5. Push to your fork: `git push origin feature/your-feature-name`
6. Open a Pull Request on GitHub and describe what you changed.

## Commit Message Convention

Use these prefixes for clean commit history:

| Prefix | When to use |
|---|---|
| `Add:` | New file or feature added |
| `Fix:` | Bug or wiring error corrected |
| `Update:` | Existing content improved |
| `Docs:` | Documentation only changes |
| `Refactor:` | Code restructured, no behaviour change |

## Reporting Issues

Open a GitHub Issue if you find:
- Incorrect pin connections
- Code compilation errors
- Documentation mistakes
- Hardware compatibility problems
